Unit testing
"[...] smallest testable part of an application."

between a class and a function

A few key points

  • Test in isolation, but not in combination
  • Runs fast, and easy to debug
  • Donâ€™t hit I/O
Integration tests
"[...] individual software modules are combined and tested as a group"

Several parts of a application tested together

A few key points

  • Test two or more parts together
  • Does test frontend and backend separately
  • Runs slower that unit tests, and harder to debug
  • Confirms that parts of your application work together
  • Can hit I/O
Unit tests vs Integration tests
Unit tests lack the bigger picture
And Integration tests still don't have the complete picture
Integration still test partially in isolation
The solution?
End to end (E2E) testing
Test the entire stack

front end and backend together

Open the application in a browser window
Happy path and smoke tests
Test core functionality
Test what you can't with unit and integration tests
